## Overview
Provides customized data table wrapper components for the ScholarConnect application. Exposes a primary factory function responsible for generating consistently styled NiceGUI tables equipped with integrated search filtering, column formatting logic, and conditionally rendered row-level action components (edit/delete operations).

## Detailed Breakdown Section

### buildDataTable

**Primary Library:** `nicegui`
**Purpose:** Constructs a styled, searchable data table with optional action buttons.

#### Overview
Assembles a complete data visualization interface. Wraps the native `ui.table` within a formatted structural card, integrates a client-side search input directly bound to the table's filter property, and dynamically calculates whether to append an 'Actions' column based on the presence of provided callback functions.

#### Workflow (Executable Logic Only)

**Phase 1: Structure and Action Resolution**
Evaluates input parameters to determine structural table needs.
* **Operation 1:** Copies `columns` to `display_columns`. Evaluates `has_actions` boolean by verifying if either `on_edit` or `on_delete` is populated.
* **Operation 2:** If actions exist, appends a synthetic column definition dict (`name: actions`, `align: center`) to house the interactive buttons.
* **Operation 3:** Initiates a `ui.card()` container with dark background (`bg-[#161b27]`), removing internal padding (`p-0`) to allow edge-to-edge table rendering.

**Phase 2: Header Assembly**
Builds the search bar and context title.
* **Operation 1:** Creates an explicit header row. Conditionally renders a `title` label or a blank `ui.space()` spacer.
* **Operation 2:** Instantiates a text input designed for search functionality, applying `bg-white/5` and `text-white/80` for low-contrast integration.

**Phase 3: Table Construction and Binding**
Wires the table component and interactive slots.
* **Operation 1:** Creates the core `ui.table()` component, passing `display_columns`, `rows`, and `row_key`. Disables initial sorting (`sortBy: None`) and sets pagination limits. Applies `.props("dark flat")` to align with the overall color scheme and remove default Quasar table borders.
* **Operation 2:** Invokes `search_input.bind_value(table, "filter")`, creating a reactive link that automatically queries the table data against the search string without additional manual callback management.
* **Operation 3:** If actions are required, attaches a custom scoped slot to `body-cell-actions`, supplying the Vue template string generated by `_buildActionSlot`.

#### Source Code
```python
def buildDataTable(
    columns: list[dict],
    rows: list[dict],
    on_edit: Callable[[dict], None] | None = None,
    on_delete: Callable[[dict], None] | None = None,
    row_key: str = "id",
    title: str = "",
) -> ui.table:
    display_columns = list(columns)
    has_actions = on_edit is not None or on_delete is not None
    if has_actions:
        display_columns.append(
            {"name": "actions", "label": "Actions", "field": "actions", "align": "center"}
        )

    with ui.card().classes(
        "w-full bg-[#161b27] border border-white/5 rounded-2xl p-0 overflow-hidden"
    ):
        with ui.row().classes(
            "w-full items-center justify-between px-6 py-4 bg-[#0f1117] border-b border-white/5"
        ):
            if title:
                ui.label(title).classes("text-white font-semibold text-base")
            else:
                ui.space()

            search_input = ui.input(placeholder="Search...").classes(
                "bg-white/5 text-white/80 rounded-lg px-3 py-1 text-sm "
                "border border-white/10 w-56"
            ).props("dense outlined dark")

        table = ui.table(
            columns=display_columns,
            rows=rows,
            row_key=row_key,
            pagination={"rowsPerPage": 10, "sortBy": None},
        ).classes("w-full text-white/80 text-sm").props("dark flat")

        search_input.bind_value(table, "filter")

        if has_actions:
            table.add_slot(
                "body-cell-actions",
                _buildActionSlot(on_edit, on_delete),
            )

    return table
```

def _buildActionSlot(
    on_edit: Callable[[dict], None] | None,
    on_delete: Callable[[dict], None] | None,
) -> str:
    edit_btn = (
        "<q-btn flat round dense icon='edit' color='indigo-4' "
        "@click=\"$parent.$emit('edit', props.row)\" />"
        if on_edit
        else ""
    )
    delete_btn = (
        "<q-btn flat round dense icon='delete' color='red-4' "
        "@click=\"$parent.$emit('delete', props.row)\" />"
        if on_delete
        else ""
    )
    return (
        f"<q-td :props='props'>"
        f"  <div class='flex gap-1 justify-center'>"
        f"    {edit_btn}"
        f"    {delete_btn}"
        f"  </div>"
        f"</q-td>"
    )
```

**Implementation (Executable Logic Only):**
* **Line 5-10:** Constructs an HTML string for a Quasar button (`<q-btn>`) only if `on_edit` is true. The button uses `@click` to emit a custom 'edit' event upwards to the parent component, transmitting the current row object `props.row`.
* **Line 11-16:** Constructs a similar string for the delete operation, emitting a 'delete' event, conditionally rendered based on the presence of `on_delete`.
* **Line 17-24:** Embeds both conditionally created buttons within a `<q-td>` cell template establishing flex layout styling for horizontal alignment.

def registerTableEvents(
    table: ui.table,
    on_edit: Callable[[dict], None] | None = None,
    on_delete: Callable[[dict], None] | None = None,
) -> None:
    if on_edit:
        table.on("edit", lambda e: on_edit(e.args))
    if on_delete:
        table.on("delete", lambda e: on_delete(e.args))
```

**Implementation (Executable Logic Only):**
* **Line 6-7:** Evaluates `on_edit`. If present, calls `.on("edit", ...)` on the table instance, intercepting the custom event broadcast from the Vue template slot and routing `e.args` (the row dictionary) into the provided callback.
* **Line 8-9:** Performs an identical evaluation and binding process for the `delete` event channel.
